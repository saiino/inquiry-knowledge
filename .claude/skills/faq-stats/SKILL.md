# FAQ Stats(月次サマリー / 効果測定)

FAQ運用がちゃんと回っているか・効いているかを git 履歴から集計するスキル。
**読み取り専用。** ファイルの変更・commit・push は行わない。

## 手順

### 1. 集計期間を決める

指定がなければ「今月(1日〜今日)」。「先月」「PoC開始から」等の指定があればそれに従う。

### 2. git 履歴から活動を集計する

`git log` を期間で絞り、コミットメッセージの規約(`faq:` プレフィックス)を使って分類する。

- FAQ 新規追加件数(`faq: FAQ-... を追加`)
- FAQ 更新件数(`faq: FAQ-... を更新`)
- 棚卸し実施回数と結果(`faq: 棚卸し ...` のメッセージから確認/修正件数を読む)
- マージまでの状況(merge コミットの有無で、PRが放置されていないか)

### 3. 現在の蓄積状況を集計する

`faq/index.md` のカテゴリ表と各ファイルから:

- カテゴリ別のFAQ件数と合計
- `最終確認日` が3か月以上前の「Microsoft仕様」エントリ数(棚卸しの滞留チェック)
- 残っているサンプルFAQの数(実FAQへの置き換わり具合)

### 4. 報告フォーマット

```markdown
## FAQ運用サマリー(YYYY-MM)

### 今期の活動
- 新規FAQ: N件(FAQ-ID列挙) / 更新: M件 / 棚卸し: K回
- 未マージのFAQ更新PR: あれば注意喚起

### 蓄積状況
- 合計N件(カテゴリ別内訳)。サンプル残りN件
- 要棚卸し(最終確認日が古い): N件

### 所感
- 運用が回っているか(報告→PR→mergeの流れが滞っていないか)を1〜3行で
- 次のアクション提案があれば1つ
```

## 測れないことを正直に言う

このスキルが測れるのは「**FAQが育っているか**」まで。「**問い合わせ対応が速くなったか**」「**未知の問い合わせの割合が下がったか**」は、問い合わせ総数を記録していないため測れない。

ユーザーが未知率や対応時間の効果を知りたがったら、次を提案する:

- 軽量な問い合わせログ(1問い合わせ1行: 日付 / カテゴリ / FAQヒット or 未知)を `log/` に置く運用。inquiry-assistant が回答案作成のついでに追記すれば、担当者の手間はゼロで未知率が測れるようになる
- ただし記録が増える分レビューのノイズにもなるため、導入するかはユーザーの判断に委ねる(勝手に導入しない)
