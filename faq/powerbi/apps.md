## FAQ-PBI-APP-001: アプリを共有したのに相手が「アクセス許可がありません」と表示される

- 登録日: 2026-07-16 / 更新日: 2026-07-16
- 種別: Microsoft仕様 / 最終確認日: 2026-07-16
- タグ: `共有` `アクセス許可` `閲覧者`
- ステータス: サンプル(動作確認用。実案件のFAQ登録後に削除してよい)

### 症状 / 問い合わせ内容

アプリのリンクを送ったが、開くと「このアプリにアクセスする権限がありません」と表示され閲覧できない。

### 原因

アプリの「対象ユーザー」に相手が含まれていない。ワークスペースに追加しただけ、またはリンクを共有しただけでは、アプリの閲覧権限は付与されない。

### 解決方法

1. ワークスペース → アプリの「アプリを更新」→「対象ユーザー」タブを開く
2. 対象のユーザーまたはセキュリティグループを追加する
3. 「アプリを更新」で再発行する(対象ユーザーの変更も再発行しないと反映されない)

### 参考リンク

- [Power BI でアプリを発行する - Microsoft Learn](https://learn.microsoft.com/ja-jp/power-bi/collaborate-share/service-create-distribute-apps)

## FAQ-PBI-APP-002: レポートを修正したのにアプリ側に反映されない

- 登録日: 2026-07-16 / 更新日: 2026-07-16
- 種別: Microsoft仕様 / 最終確認日: 2026-07-16
- タグ: `アプリ更新` `反映` `発行`
- ステータス: サンプル(動作確認用。実案件のFAQ登録後に削除してよい)

### 症状 / 問い合わせ内容

ワークスペースのレポートを修正したが、利用者が見ているアプリの画面が古いままになっている。

### 原因

ワークスペース上のレポート修正は、アプリを再発行(「アプリを更新」)するまでアプリには反映されない。ワークスペースとアプリは別のスナップショットを持つ。

### 解決方法

1. ワークスペースで「アプリを更新」を実行する
2. データ自体が古い場合は、セマンティックモデルの更新(スケジュール更新または手動更新)も確認する

### 参考リンク

- [発行されたアプリを更新する - Microsoft Learn](https://learn.microsoft.com/ja-jp/power-bi/collaborate-share/service-create-distribute-apps#update-your-published-app)
