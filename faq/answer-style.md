# 回答スタイルガイド

inquiry-assistant が回答案を作るときの文面の型。
**担当者ごとの文体のばらつきを無くし、誰が対応しても同じトーンで返すためのガイド。**
チームの文化に合わせて自由に書き換えてよい(このファイル自体もPRで育てる)。

## 宛先の想定

社内の一般利用者(Power Platform に詳しくない前提)。

## 基本構成

1. **書き出し**: 定型の挨拶(下記)
2. **結論**: 原因と対処を最初の2〜3行で言い切る(詳細より先)
3. **手順**: 番号付き。1手順1操作。画面上の表記は「」で正確に書く(例: 「アプリを更新」)
4. **依頼事項**(あれば): 利用者にやってほしいこと・教えてほしいことを明示
5. **締め**: 定型の結び(下記)

## 定型文

- 書き出し: `お問い合わせありがとうございます。Power Platform保守担当です。`
- 締め: `お試しいただき、解消しない場合はその旨ご返信ください。`
- 調査に時間がかかる場合の一次回答: `確認にお時間をいただいております。◯日までに状況をご連絡します。`

## トーンのルール

- です・ます調。過度な謝罪はしない(サービス仕様による挙動は謝る事柄ではない)
- 専門用語には一言説明を付ける(例: 「セマンティックモデル(レポートの元データ)」)
- 断定できないことは断定しない。「〜の可能性があります。確認のため◯◯を教えてください」の形にする
- 利用者の操作ミスが原因でも、責める表現をしない(「〜という仕様のため」と仕様側の説明にする)

## NG例 → OK例

- ×「権限がないからです」 → ○「アプリの閲覧権限が付与されていない状態のためです」
- ×「Learnを見てください」 → ○ 手順を本文に書き、参考としてリンクを添える
- ×「たぶんライセンスの問題です」 → ○「ライセンス起因の可能性があります。お使いのライセンス種別を確認させてください」
