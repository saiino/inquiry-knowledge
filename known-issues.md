# 既知障害・一時情報(known issues)

**「今だけ正しい情報」の置き場。** FAQ(恒久ナレッジ)とは分けて管理する。
Microsoft側の障害・計画メンテ・社内側の一時的な設定変更など、同じ問い合わせが短時間に集中する事象をここに書く。

## 運用ルール

- inquiry-assistant は**FAQより先にこのファイルを確認**し、該当すれば「既知の事象」として即回答案を出す(個別調査をしない)
- 速報性を優先するため、**このファイルだけはPRレビュー不要・直接コミットでよい**(履歴はGitに残る)
- 解消したらエントリを削除する(消した記録もGit履歴に残るので、後から「あの日の障害」を振り返れる)
- 恒久的な知見が得られた場合(回避策が今後も使える等)は、削除時に faq-updater でFAQ化を検討する

## エントリのテンプレート

```markdown
## 🔴 対応中: <事象の一行説明>

- 発生: YYYY-MM-DD HH:MM / 情報源: (M365管理センターのインシデントID、社内連絡など)
- 影響: (誰の何ができない・遅いのか)
- 問い合わせへの回答方針: (個別調査の要否、利用者に伝える内容)
- 回避策: (あれば)
```
